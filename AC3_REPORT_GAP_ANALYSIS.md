## 6. Critical Gaps Requiring Pipeline Fixes

### Gap 1: No engagement timeline events
The `engagement_timeline_events` table has zero rows for Vianova. The current `importEngagementFindings` procedure relies on this table. **The primary data source must be `engagement_ops_snapshots.state_json.vulnAnalysis`**, not timeline events.

**Fix:** Add a new import path that reads directly from `engagement_ops_snapshots.state_json` — extracting `vulnAnalysis` entries as findings, `attackChains` for ATT&CK mappings, and `essIntelligence` for CVE enrichment.

### Gap 2: ATT&CK technique IDs not auto-extracted per finding
Attack chains contain `techniqueId` per step, but these are not mapped to individual vulnerability findings. The `vulnAnalysis` entries do not carry `techniqueId` directly.

**Fix:** During import, cross-reference `vulnAnalysis[].finding.asset` and CVE IDs against `attackChains[].killChainPhases[].steps[]` to associate technique IDs with each finding. Also use a static ATT&CK-to-technique lookup for common CVE categories.

### Gap 3: NIST 800-53 control mappings not present
No engagement data carries NIST control IDs. The AC3 report schema requires `controls` per finding.

**Fix:** Implement a static mapping table from ATT&CK technique IDs to NIST 800-53 controls (e.g., T1190 → SC-7, SI-4, RA-5, CA-8). Apply this mapping automatically during import.

### Gap 4: CVSS scores/vectors not parsed
The `essIntelligence.topThreats` array contains CVSS scores embedded in `riskSummary` strings but not as discrete fields.

**Fix:** Parse CVSS scores from `riskSummary` strings during import. For CVEs not in `topThreats`, use the `vulnAnalysis[].analysis.riskScore` (1-10 scale) as a proxy.

### Gap 5: Scope fields not auto-populated
Authorized domains, crown jewels, and approved vectors are available in the ops snapshot but not auto-populated into the AC3 report during creation.

**Fix:** When importing from an engagement, auto-populate `rpt_scope_domains` from `roeScopeGuard.authorizedDomains`, `rpt_scope_assets` from `engagementContext.crownJewels` + asset hostnames, and `rpt_cloud_provider` from `assets[].cloudServices`.

### Gap 6: Evidence not auto-assembled
PoC text, exploit output, and approval gate details are available but not assembled into the `rf_evidence` JSON array per finding.

**Fix:** During import, compile evidence from: (a) `vulnAnalysis[].analysis.poc`, (b) matching `assets[].exploitAttempts[].exploitOutput`, (c) relevant `approvalGates[].description`.

### Gap 7: Executive summary inputs not auto-derived
Detection opportunities and impact assessments exist in attack chains and vuln analysis but are not fed into the executive summary generation.

**Fix:** During import, extract `attackChains[].detectionOpportunities` as `rpt_exec_strengths` seed data, and aggregate high/critical `vulnAnalysis[].analysis.impactAssessment` as `rpt_exec_gaps` seed data.
